# Implementation Plan: Learn by Doing

**Created:** 2026-03-24 **Type:** Feature **Overview:** A `/learn-by-doing` skill — an alternative to `/build` that uses the same implementation plan but flips the roles. The user writes all the code; Claude acts as a tutor using evidence-based learning techniques forked from the learning-opportunities skill (CC-BY-4.0, Dr. Cat Hicks). **Design Spec:** docs/design-specs/2026-03-24-1616-learn-by-doing.md

---

## Summary

Create a `/learn-by-doing` skill that consumes the same implementation plan as `/build` but inverts the roles — the user writes all code while Claude facilitates learning using evidence-based techniques forked from the learning-opportunities skill. Includes task-by-task progression with learning techniques, stuck escalation, build log, learning log, and commit checkpoints.

### Task 2: Create SKILL.md for /learn-by-doing

**Description:** Create the main skill file with frontmatter (`disable-model-invocation: true`, `allowed-tools: Read, Grep, Glob`) combining:

- Build-style workflow structure (task-by-task progression, build log entries, commit checkpoints, pause between tasks, acceptance criteria walkthrough, phase complete announcement)
- Announce location matching `/build` format: `**Phase 3: Learn by Doing** | Task [N]/[Total]: [Task Name]`
- Learning facilitation forked from learning-opportunities (exercise types, pause-for-input protocol, facilitation guidelines, techniques to weave in, hands-on code exploration / fading scaffolding)
- Stuck escalation flow (hint → pointed question → point to pattern → show answer) as a safety net after learning techniques have been tried
- Learning log created at skill invocation at `docs/learning/YYYY-MM-DD-HHMM-feature-name.md` with sections: technique used per task, what clicked, gaps surfaced, revisit later
- User writes all code; Claude never writes implementation code directly
- CC-BY-4.0 attribution for learning-opportunities

Use `/build` SKILL.md as the structural skeleton and weave in learning-opportunities content for the pedagogical layer.

**Carry forward from `/build`:** Announce, Verify, Report, Log, Commit, Pause steps; Handling Deviations section (verbatim); Phase Complete announcement; all Rules except #2.

**Adapt:** Step 2 "Implement" → Claude selects a learning technique and facilitates; user writes the code. Rule #7 "No git operations" → user still handles git (same rule, but now also handles all code writes).

**Add:** Learning facilitation layer between Announce and Verify. Stuck escalation as a fallback within the facilitation step. Learning log entry after each task's Pause step.

## Acceptance Criteria

- [ ] `/learn-by-doing` is invocable as a skill and loads the implementation plan
- [ ] Claude selects and applies a learning technique per task (exercise types from learning-opportunities)
- [ ] Claude pauses for user input during exercises — no hints, no suggested answers
- [ ] User writes all implementation code; Claude does not write code for the user
- [ ] When user is genuinely stuck, escalation sequence activates (hint → pointed question → point to pattern → show answer)
- [ ] Build log in implementation plan is filled in identically to `/build`
- [ ] Learning log is created at `docs/learning/YYYY-MM-DD-HHMM-feature-name.md`
- [ ] Commit checkpoints and pause-between-tasks work the same as `/build`
- [ ] `/document` phase works identically after `/learn-by-doing` as after `/build`
- [ ] learning-opportunities attribution (CC-BY-4.0) is present in the skill file
